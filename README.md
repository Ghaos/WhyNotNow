# WhyNotNow

WhyNotNow は、Codex の会話ワークベンチと連携する「後回しにした作業」の受信トレイです。ブラウザまたはCodexで気になる作業を記録し、なぜ今ではないのかを掘り下げ、別の AI 応答を待つことなく見直しや完了にできます。

## はじめる

スキルを短いメモとともに呼び出します。

```text
$wnn オンボーディングメールを簡潔にすべきか調べる
```

WhyNotNow はアイデアを記録し、選択フォームを出さずに「なぜ今ではないのか」の対話を始めます。保存後はダッシュボードから次の2方式で開始できます。

- **Do it now** は、ここまでに集めた文脈を引き継ぐ Codex チャットを開きます。最初のターンではまだ実行せず、そのチャットが見えている状態で **開始** と返信した後、同じチャットで実行します。
- **Why not now?** は、あなたが挙げた懸念を起点に進みます。その点を明確にしたり、関連する条件と結び付けたり、分かっていることを要約したりします。障害に小さく具体的な解決の糸口がある場合は、もう一度判断する前にその調査を提案できます。

メモには、作業が重要な理由、分かっている制約、関連リンクも含められます。あとから文脈を追加することもできます。WhyNotNow は理由を列挙するよう求めるのではなく、会話に沿って進みます。

## 受信トレイを開く

Codex と WhyNotNow プラグインの実行中に、ブラウザで [http://127.0.0.1:49321/](http://127.0.0.1:49321/) を開きます。ダッシュボードは自動更新され、タスクを **実行前**、**実行中**、**完了** の3タブで表示します。上部のフォームにタスク本文だけを書いて追加することもできます。

- 実行前項目の **Why not now?** は、選択済みのCodex対話を開いて障害の整理を始めます。
- **Do it now** は準備済みのCodexチャットを開きます。そのチャットで **開始** と返信すると実行先として紐付き、実行中へ移ります。
- Codexへの遷移を試した起動用ウィンドウは自動で閉じます。
- 実行中項目の **Codexで開く** から、紐付いた実行チャットへ戻れます。
- どの状態でもチェックすると完了にでき、チェックを外すと直前の実行前または実行中へ戻ります。

ダッシュボードは、保留項目を追加し、状態別に確認し、開始方式を選び、完了するための主な場所です。Codex は、相談、編集、調査、実作業、アーカイブを行う場所として使い続けます。

この二つは別々の管理画面ではなく、「ブラウザまたはCodexで記録し、保留箱で確認し、必要なら Codex で整理する」という一つの循環です。操作をどちらへ置くかの設計原則は、[AI と GUI の境界](docs/interaction-boundary.md)を参照してください。

## 期待できること

WhyNotNow は、一度に関連する質問を 1 つだけ尋ねます。回答のたびに機械的に次の理由を尋ねることはありません。特定の障害を手伝える場合は、実施できる最小限の読み取り専用調査を先に説明し、承認を求めます。ダッシュボードから開く場合、元の作業は **Do it now** を選び、表示されたCodexチャットで **開始** と返信するまで開始されません。

## アイデアに戻る

保留項目の一覧確認はブラウザのダッシュボードで行います。`$wnn` では、個別に保存された会話を表示、追記、編集、再訪、完了、復元、開始、アーカイブできます。各アイデアは分かれているため、無関係な作業が混ざることはありません。

## プライバシー

WhyNotNow の会話はデバイス上に保存されます。WhyNotNow はテレメトリーやクラウド同期を使用しません。

初回利用時には、会話を保存するための限定的な権限を Codex が求めることがあります。WhyNotNow に対して表示された権限だけを承認してください。
